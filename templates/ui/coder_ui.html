Questions Assigned to You

You have been assigned questions to code.

You have been assigned questions to code. Each question has a list of about 50 characteristics that need to be coded. For each characteristic there is either a list of choices or an entry field for a numeric value. In addition there is a text that explains the details about each characteristic that may be used to clarify what the characteristic is asking about. You will need to complete each characteristic and decide the correct values. You will be informed when the coding for each question is complete and then you may move on to the next question in the assignment.

Frequent Asked Questions while Coding

1. Which texts are present in SQP?

2. How to code Domain and Concept characteristics?

3. How to code Social desirability? And Centrality?

4. How is SQP treating batteries of questions?

5. How do I code the Don’t know option?

6. How do I code the Interviewer instructions?

7. How do I code the Show Cards characteristics?

1. Which texts are present in SQP?

It is very important to bear in mind that in SQP only the text that is read to or by the respondent will be present. Thus, such texts are the Introduction (if used), the Request for an answer and the Answer options. This means that all instructions to the interviewer or everything, which is not read, to or by the respondent should not be present in the SQP text. However, there will be characteristics that will identify this extra information during the coding process. This is why we suggest having the real questionnaire in hand to detect this missing information in SQP.

2. How to code Domain and Concept characteristics?

The Domain and Concept characteristics are usual to present uncertain decisions. There are many different domains and concepts among which to chose, and the coder can be confused. In order to get the complete definition of each we suggest using the SQP Help screens or the SQP Coding Guideline. The coding is done in two steps and sometimes a better code can be found by choosing another first category. For further explanation we refer to Chapter 2 from Saris and Gallhofer 2007.

3. How to code Social desirability? And Centrality?

The coder must be aware that the Social Desirability characteristic will be specific for the population under study, so cultural and time references must be taken into account. For example, if a coder decides to code a question from ESS Round 1 in Spain he or she must take into account that the survey took place in 2002 and that the respondents where the Spanish population. The sensitivity over questions about political interest may change across time. The same happens with the Centrality characteristic; the coder must place on the mind of the respondents and think if the topic can be central or not in the mind of the survey respondents (e.g. in the mind of the Spanish population in 2002).

4. How is SQP treating batteries of questions?

SQP has a special form to treat the batteries of questions. The request and if applicable the introduction is placed just before the first statement. One has to think of it as the sequence of how it is read to the respondent.

Example: Battery format in questionnaire ESS Round 6. In ESS Round 6 questions E17 to E21 ask about democracy in the country.

The presentation text in SQP for the questions presented in the battery will be as follows:

The understanding of this procedure will matter for the coding of the characteristics: Formulation on the request for an answer and Use of stimulus or statement in the request.

Following the example, we will say that question E17 is formulated as an indirect request (“Using this card, please tell me to what extent you think each of the following statements applies in [country]…”) with a stimulus or statement present (“National elections in [country] are free and fair”). However, questions E18 to E21 will be coded in the characteristic Formulation of the request for an answer as ‘No request present’ and with a stimulus or statement present (i.e. for E18 the statement is “Voters in [country] discuss politics with people they know before deciding how to vote”, for E19 it is “Different political parties in [country] offer clear alternatives in one another”, and so on).

5. How do I code the Don’t know option?

There are three options to deal with the Don’t know option in a question. The first is to explicitly provide it to the respondent, in this way the don’t know option will be treated as another answer option and will be present in SQP text. However, many studies apply the implicit don’t know option, which is not mentioned to the respondent as an answer option but can be chosen by the interviewer if the respondent does not know how to answer. In this case, it will not appear in the SQP text, however, the coder must code this characteristic as “Don’t know option only registered”. The third case in when the answer options are provided with no don’t know option.

6. How do I code the Interviewer instructions?

The characteristic Interviewer instruction is only present if an interviewer is reading the questionnaire to the respondent. These instructions are not present in SQP because they are not read to the respondent. However, if an interviewer administrated the survey the coder must check the presence of this kind of instructions in the questionnaire.

7. How do I code the Show Cards characteristics?

The Showcard characteristics will imply an interviewer instruction regarding the use of a show card for a question. It will not appear in the SQP text but if a Show card was used the coder will need to code its characteristics (i.e. horizontal or vertical scale, picture provided, numbers or letters before answer options, etc…).
